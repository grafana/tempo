# Tempo → wiresmith protobuf compiler migration

Tracking document for replacing protoc-gen-gogofaster with
[wiresmith](https://github.com/grafana/wiresmith) for Tempo's own protos.
Wire format is unchanged (standard protobuf, binary-compatible with gogo);
only the generated Go API shape changes.

Toolchain: `wiresmith` CLI (v0.0.0-20260609114539-bee7aa4), invoked from
`make gen-proto`. `go.mod` carries `require github.com/grafana/wiresmith`
(runtime `protohelpers` package) with a local `replace` while the module is
unpublished.

## Migration notes (patched OTel protos)

- Repeated message fields annotated `(wiresmith.options.pointer) = true` to
  preserve gogo's `[]*T` shapes and limit call-site churn. Singular message
  fields inside oneofs become values (oneof wrappers hold values).
- `AnyValue.String()`: gogo emitted proto-text; wiresmith emits
  `fmt.Sprintf("%v", *m)` which prints interface pointer addresses for oneof
  fields and is NOT deterministic. A hand-written `StableString()`
  (pkg/tempopb/common/v1/common_gogo_shim.go) provides a stable identity;
  `spansetID` uses wire-bytes identity instead.
- gogo's reflection-based `proto.Equal` / `proto.Clone` / `proto.Merge`
  cannot traverse wiresmith structs (unexported `fieldsPresent [1]uint64`
  bitmap; gogo only skips `XXX_`-prefixed fields). Fallout fixed:
  - `test.ProtoEqual` (pkg/util/test/req.go) compares marshaled wire bytes;
    all `proto.Equal` call sites on trace-bearing types switched to it.
  - `modules/frontend/combiner/common.go` clones responses via a
    Marshal/Unmarshal round trip (`cloneProto`) instead of gogo
    `proto.Clone`. Side effect: empty top-level slices come back nil
    (wire-equivalent; tests updated).
  - `cmp.Diff` comparisons need `cmpopts.IgnoreUnexported(common_v1.KeyValue{})`
    (tempodb/tempodb_metrics_test.go).
- gogo `jsonpb` still works for Trace JSON because the shim provides
  `XXX_OneofWrappers` and registers enums with the gogo registry
  (`*_gogo_shim.go` files).

## Wiresmith blockers

Confirmed blockers / friction points, ranked. Each entry: where it bites,
what gogo offered, why wiresmith can't express it, suggested feature,
severity.

1. **Generated `String()` is not deterministic and not overridable**
   - Proto/field: `tempopb.common.v1.AnyValue` (any message with a oneof).
   - gogo: `String()` emitted proto-compact-text — deterministic, no
     pointer addresses.
   - wiresmith: emits `fmt.Sprintf("%v", *m)` unconditionally; a
     hand-written override in the same package is a compile error
     (duplicate method).
   - Suggested: emit a deterministic text form for oneof variants (or an
     option to suppress `String()` generation so consumers can supply one).
   - Severity: medium (worked around with `StableString()` + call-site
     changes; silent nondeterminism is easy to miss).

2. **Unexported presence bitmap breaks gogo reflection interop
   (`proto.Equal/Clone/Merge`, `cmp.Diff`)**
   - Any wiresmith message embedded in a still-gogo graph during a staged
     migration.
   - gogo skips `XXX_`-prefixed fields; the earlier wiresmith bitmap name
     (`XXX_fieldsPresent`) interoperated, the current unexported
     `fieldsPresent` does not — gogo logs `proto: don't know how to
     compare/copy` and returns false / drops data.
   - Suggested: document loudly, or restore an `XXX_`-style exported name
     behind a compat flag for mixed-graph migrations.
   - Severity: medium-high for staged migrations (every `proto.Equal` call
     site had to change); zero once a graph is fully wiresmith.
